    单独调用：方法名（）；
    打印调用：System.out.println(方法名（参数若干）)；
    赋值调用：数据类型 变量名 = 方法名称（参数）；

  • 方法的重载（OverLoad）
    方法名称一样，参数列表不一样；

方法的重载与下列因素有关：参数个数不同；参数类型不同，参数类型的位序不同；
方法的重载与下列因素无关：方法名称；方法的返回值类型；方法的修饰符

  1. 数组

  • 数组是一种引用数据类型

  • 数组的创建：
    （1）动态初始化（指定长度）数据类型[] 数组名称 = new 数据类型[数组长度]；
    （2）静态初始化（指定元素）数据类型[] 数组名称 = new 数据类型[]{元素1，元素2...}；
    简略写法：数据类型[] 数组名称 = {元素1，元素2...}；
    一些示例：

  • 数组名称–数组内存地址哈希值。

  • 在进行动态初始化的时候，数组初始默认值如下：
    整型:0
    浮点型：0.0
    字符型：‘\u0000’—（这是一个不可见字符，不是空格）；
    布尔型：false
    引用类型：null

  • java的内存分配
    （1）栈（Stack）：存放方法中的局部变量。局部变量：方法的参数，方法内的变量。只在作用域内起作用。
    （2）堆（Heap）:new出来的放在堆内存中，都有一个16进制的地址值，堆内存中的数据都有一个默认值。
    （3）方法区（Method Area）：存储.class文件相关信息，包含方法相关信息。
    （4）本地方法栈（Native Method Stack）:与操作系统相关。
    （5）寄存器（pc Register）:与CPU相关。

  1. 面向对象

  • 面向对象&面向过程

  • 类

  • 类的创建

（1） 导包
指出要使用的类的位置：
import 包名.类名；
如果和当前类在一个包下，则可以省略。

（2） 创建
格式：类名 对象名 = new 类名（）；
Student stu = new Student();

（3）使用：stu.成员变量名； stu.成员方法（参数）；

  • 内存的使用情况：


    堆内存中的对象中的成员方法保存的是方法区的方法的地址；
    对象名是该对象在堆内存中的地址；
  • static ：静态方法，不需要对象就能够使用
  • 对象名也是一种引用类型，可以作为参数或者返回值；
  • 局部变量和成员变量的区别：

（1）定义的位置不一样：
方法的内部，方法的外部；

（2）作用范围：
只在方法内，整个类中都可以使用；

（3）默认值：
局部变量没赋值就不能使用，成员变量有默认值：
String：null
int：0
double：0.0
boolean：false

（4）内存位置：
栈内存，堆内存

（5）生命周期：
局部变量随着方法的入栈而诞生，随着方法的出栈而释放
成员变量随着对象的创建而诞生，随着垃圾回收而消失

  1. Arraylist类

  • 数组的长度不可变，ArrayList的长度可变；

  • ArrayList E称为泛型，E只能为引用类型，不能为基本数据类型；

  • 格式：ArrayList<String> list = new ArrayList<>();

  • 对于集合来说，直接打印的结果是内容，而不是地址值；
    System.out.println(list);//[]

    集合中的常用方法：

  • 添加元素public boolean add(E e);//添加元素e
    注：对于ArrayList集合来说，add操作一定是成功的，返回值可用可不用，但是对于其他集合来说不一定；

  • 读取元素public E get(int index);//读取下标为index的元素

  • 删除元素public E remove(int index);//读取下标为index的元素，并返回这个元素

  • 获取集合长度public int size();获取集合的长度

ArrayList集合中存放基本类型，需要使用基本类型对应的包装类：

  • 基本类型 包装类型
    byte Byte;
    short Short;
    int Integer;(*)
    long Long;
    char Character;(*)
    boolean Boolean;
    double Double;
    float Float;
    格式：Arraylist<Integer> list = new ArrayList<>();
    注：JDK 1.5+支持自动装箱和自动拆箱；
    自动装箱：自动将基本类型转换为其包装类；
    自动拆箱：自动将包装类型转化为其对应的基本类型；
